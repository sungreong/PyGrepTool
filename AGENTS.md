# AGENTS.md

## 작업 원칙

- 변경 전에는 관련 코드와 기존 테스트를 먼저 읽고, 가능한 경우 기준선 테스트를 실행한다.
- 프로젝트에 적용 가능한 스킬이 있으면 해당 `SKILL.md`를 끝까지 읽은 뒤 사용한다. 스킬이 맞지 않으면 억지로 적용하지 말고 일반 코드 작업으로 진행한다.
- 현재 동작이 잘 되는 코드는 작은 개선만 한다. 공개 API, CLI 출력, tool schema처럼 사용자가 의존할 수 있는 표면은 테스트 없이 바꾸지 않는다.
- 로컬 파일 검색은 `rg`를 우선 사용하고, 없을 때만 다른 방법을 쓴다.

## 1000줄 이내 코드 작성 규칙

- 사람이 직접 관리하는 소스 파일은 파일당 1000줄 이내를 원칙으로 한다.
- 한 파일이 800줄을 넘기기 시작하면 기능 단위로 나눌 수 있는지 먼저 검토한다.
- 1000줄을 넘겨야 하는 경우에는 생성물, 큰 fixture, 외부 스키마처럼 분리 비용이 더 큰 파일인지 확인하고 이유를 주석이나 문서에 남긴다.
- 함수와 클래스도 한 파일에 몰아넣지 말고, backend, CLI, tool handler, test helper처럼 책임 단위가 보이면 모듈로 분리한다.
- 분리할 때는 public API와 기존 import 경로를 가능한 한 유지하고, 변경이 필요하면 테스트와 README 예시를 함께 갱신한다.

## 테스트 규칙

- 코드 변경 후에는 최소 `python -m pytest`를 실행한다.
- 버그 수정이나 동작 개선은 먼저 실패하거나 부족한 케이스를 테스트로 표현한 뒤 구현한다.
- 외부 명령어 의존성이 있는 `rg`, `grep` 관련 변경은 순수 Python backend 테스트도 함께 확인해 fallback 동작을 보호한다.
- tool 결과 JSON의 필드 의미가 바뀌면 `tests/test_tool.py`에 직렬화와 edge case 테스트를 추가한다.
